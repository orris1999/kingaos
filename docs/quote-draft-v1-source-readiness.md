# Quote Task 003E｜V1 数据源准入范围与 adapter 修正清单

日期：2026-05-12

本文件基于 `docs/quote-source-dry-run-report-2026-05-12.md` 的脱敏 dry-run 结果，锁定 `KJ 批量报价草稿` V1 的数据源准入范围。本文只做准入判断和实施前 checklist，不读取真实 Excel，不导入数据库，不生成报价草稿，不生成正式报价。

Quote Task 003E-R 已根据会议确认补充水箱 / 中冷器规则，详见 `docs/quote-draft-radiator-intercooler-rules.md`。Quote Task 004B-R 进一步修正口径：水箱 / 中冷器不是整个品类都必须人工确认；它们的主成本表可以进入 V1，完整标准 KJ 唯一匹配时可进入草稿，只有命中旧码、鼎捷码、OEM、多候选、特殊 sheet、风险字段、包装规格不明确或 Excel 嵌入图等具体行级风险时，才提示人工确认。

## 业务边界

- 报价表 / 成本表 / 价格候选数据由财务提交和维护。
- 出口部不能上传报价表，不能维护价格表，不能设置底价或毛利。
- 出口部只能消费财务提交的数据生成报价草稿。
- 报价草稿不是正式报价，不能发客户。
- 成本报价表只能作为 `priceCandidate` / `costCandidate` 来源候选，不是财务批准价格。
- V1 不自动批准价格，正式报价必须后续接入 FinancePricing。

## V1 数据源准入结论

### A. 可进入 V1 KJ 批量报价草稿

这些表在 003D dry-run 中均满足：有 KJ 列、有产品名称或可展示名称列、有成本候选列、adapter confidence = `high`，且结构相对稳定。V1 仍只允许生成内部报价草稿候选，价格状态必须标记为非财务批准价格。

| 品类 | adapterId | 准入结论 | V1 处理要求 |
|---|---|---|---|
| 冷凝器 | `condenser-cost-2026` | 可进入 V1 | 处理标准 / ERP / 孚盟等多 KJ 来源；图片只显示 `embedded_only` 状态。 |
| 暖风 | `heater-cost-2026` | 可进入 V1 | 明确 `KJ总编码` 和 KJ 变体列优先级；保留自产 / 外购属性。 |
| 蒸发器 | `evaporator-cost-2026` | 可进入 V1 | 处理多 KJ 列冲突；无稳定图片来源时显示缺图状态。 |
| 水室 | `water-chamber-cost-2026` | 可进入 V1 | 按 KJ 关联左右水室结构；草稿中必须提示这是左右件结构，不是通用整品表。 |
| 全铝自产机冷 | `all-aluminum-oil-cooler-cost-2026` | 可进入 V1 | 需要从主件品名字段提取 KJ；作为单独产品线规则验收。 |

### B. 可进入 V1，但带条件和行级人工确认规则

这些表有 KJ、产品名称、成本候选列，adapter confidence = `high`，主成本表可以进入 V1。它们不是整类人工确认；V1 应先输出 `v1_eligible_with_conditions`，再按具体行风险决定是否显示 `需人工确认`。

| 品类 | adapterId | 风险 | V1 处理要求 |
|---|---|---|---|
| 水箱 | `radiator-cost-2026` | sheet 多，包含 OEM / OE、限销、不能生产、只做报价不公布、包装方案等复杂信息。 | 主 sheet `2026年 水箱成本报价表` 可作为 V1 主草稿候选；完整标准 KJ 唯一匹配可进入草稿；基础 KJ、多候选、旧码、鼎捷码、OEM、特殊 sheet、风险字段或包装规格不明确时提示人工确认。 |
| 中冷器 | `intercooler-cost-2026` | 与水箱结构相近但字段和数据量不同，不能简单复用水箱结论。 | 主 sheet `2026年 中冷器成本报价表` 可作为 V1 主草稿候选；完整标准 KJ 唯一匹配可进入草稿；基础 KJ、多候选、旧码、鼎捷码、OEM、特殊 sheet、风险字段或包装规格不明确时提示人工确认。 |

水箱 / 中冷器辅助 sheet 处理规则：

- `不能生产`、`只做报价，不公布的报价表`、`徐芳芳 不保质，漏水不赔，需与客户说明情况` 以及类似风险说明 / 特殊用途 sheet 不进入 V1 主草稿。
- 这些 sheet 未来只作为内部风险参考；如果命中相关 KJ，需要人工确认，不自动生成报价草稿。
- `KJ-编码（标准编码）` 是 V1 标准 KJ；`旧 KJ.NO` / 孚盟编码、鼎捷编码（带水箱盖 / 不带水箱盖）必须保留为历史引用或生产下单参考。
- 基础 KJ 可能对应多候选，不能静默选择第一行。
- `2026.5.11出口成本报价` 和 `2026.5.11出口部内销成本报价` 分别是外销 / 内销成本候选，不是财务批准价格；历史日期列不作为 V1 默认候选。

### C. 不进入 V1 产品 KJ 报价草稿，只作为包装 / 附加项候选

| 品类 | adapterId | 结论 | 原因 |
|---|---|---|---|
| 特殊包装及其他 | `special-packaging-cost-2026` | 不进入产品 KJ V1，只作为包装 / 附加项候选 | dry-run 未检测到稳定 KJ / OEM 主键；表性质更像包装 / 其他成本补充，不能直接视为产品标准报价。 |

### D. 暂缓

003D dry-run 中没有必须整体暂缓的产品类 adapter。所有产品类 adapter 的匹配置信度均为 `high`。但以下能力仍暂缓，不进入 V1：

- OEM / OE 自动匹配。
- Excel 嵌入图片作为稳定主图。
- 特殊包装及其他作为产品行。
- 正式报价、财务批准价格、底价 / 毛利计算。
- 自动导出正式 PDF / Excel 或发送客户。

## V1 实施前 checklist

上线 V1 开发前，必须完成以下确认：

1. 财务确认哪几张表进入 V1：建议首批为冷凝器、暖风、蒸发器、水室、全铝自产机冷，水箱 / 中冷器以 `v1_eligible_with_conditions` 进入。
2. 财务确认价格字段含义：成本候选、报价候选或其它，且字段含义写入 adapter 配置。
3. 财务确认所有价格都不是 `FinanceApprovedPrice`。
4. 出口部确认 V1 只生成报价草稿，不生成正式报价，不发客户。
5. 技术 / 产品资料负责人确认图片来源策略：Excel 嵌入图片不作为稳定主图来源。
6. 产品主数据负责人确认 KJ 标准化规则：标准编码、旧编码、ERP 编码、孚盟编码的优先级和冲突处理。
7. 水箱 / 中冷器多编码、多包装、多规格映射需要人工确认规则。
8. 特殊包装及其他作为包装 / 附加项候选，不作为产品行。
9. 正式报价审批仍需 FinancePricing，V1 不自动批准价格。
10. 任何导入生产库、staging table、财务上传页面或正式报价链路都必须另立任务并重新评审。
11. 水箱 / 中冷器数量字段必须保留，数量缺失时输出 warning，未来可用于体积 / 是否够柜计算。

## V1 验收标准调整

V1 只允许做到：

1. 输入 KJ + 数量。
2. 系统生成报价草稿行。
3. 显示产品名称或可展示名称。
4. 显示品类。
5. 显示图片状态。
6. 显示 `priceCandidate` / `costCandidate` 状态。
7. 显示 warnings，包括成本候选、缺图、嵌入图、多编码、多包装、多规格和需人工确认。
8. 标明“非正式报价”。
9. 标明“价格候选不是财务批准价格”。
10. 不允许进入 `sent_to_customer`。

V1 不允许做到：

1. OEM 自动匹配。
2. 正式报价。
3. 价格审批。
4. 底价 / 毛利计算。
5. 导出正式 PDF。
6. 自动发送给客户。
7. 销售上传或维护价格表。
8. 销售自行决定价格是否可用。

## Go / No-Go 建议

Go：

- 冷凝器、暖风、蒸发器、水室、全铝自产机冷可作为 V1 产品 KJ 草稿数据源候选。
- 水箱、中冷器主成本表可进入 V1；完整标准 KJ 唯一匹配可进入草稿，行级风险再触发人工确认。

No-Go：

- 特殊包装及其他不进入产品 KJ V1。
- OEM / OE 自动匹配不进入 V1。
- 任何成本候选都不能被命名或展示为财务批准价格。
- 任何草稿结果都不能发客户或进入正式报价状态。
