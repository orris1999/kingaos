# Quote Task 003E-R｜水箱 / 中冷器规则确认结果

日期：2026-05-13

本文件固化会议确认后的水箱 / 中冷器规则。结论不是“水箱 / 中冷器不能做”，而是“不能在 V1 中静默自动匹配”。它们可以进入 KJ 批量报价草稿方向，但必须带人工确认、风险提示、价格边界和字段解释。

## 总体结论

- 水箱 / 中冷器可以进入 V1 KJ 批量报价草稿方向。
- 水箱 / 中冷器主成本表可以进入 V1，不是整个品类都必须人工确认。
- 人工确认是行级别 / 情况级别：完整标准 KJ 唯一命中主成本表时，可以进入 V1 草稿；基础 KJ、多候选、旧码、鼎捷码、OEM、特殊 sheet、风险字段或包装规格不明确时，才提示需人工确认。
- V1 必须只做 KJ 草稿候选，不生成正式报价。
- V1 必须显示风险提示和字段解释。
- OEM / OE 自动匹配暂不开放。
- 价格候选不是财务批准价格，正式报价必须后续接 FinancePricing。

复杂原因：

1. sheet 不止一个。
2. KJ 编码来源多。
3. OEM / OE 出现在辅助 sheet，不适合 V1 自动匹配。
4. 包装和规格字段多。
5. 状态和风险字段多。
6. 价格字段名称容易误解。
7. 历史日期价格列并存。
8. Excel 图片不是稳定资产。

## Sheet 使用规则

### 进入 V1 主草稿候选的 sheet

水箱主 sheet：

- `2026年 水箱成本报价表`

中冷器主 sheet：

- `2026年 中冷器成本报价表`

用途：

- 作为 V1 KJ 批量报价草稿的主数据来源候选。
- 只生成报价草稿。
- 不生成正式报价。
- 价格只作为 `priceCandidate` / `costCandidate`。

### 不进入 V1 主草稿的 sheet

以下 sheet 不进入 V1 主草稿：

- `不能生产`
- `只做报价，不公布的报价表`
- `徐芳芳 不保质，漏水不赔，需与客户说明情况`
- 类似风险说明 / 特殊用途 sheet

处理方式：

1. 不直接给出口部自动消费。
2. 不作为主报价行来源。
3. 可作为未来内部风险参考。
4. 如果未来命中相关 KJ，需要人工确认。
5. 不自动生成报价草稿。

## KJ / 编码规则

### 字段含义

| 字段 | 含义 | V1 处理 |
|---|---|---|
| 编码 | 基础 KJ / base KJ，可能不带完整规格，可能对应多行。 | 可保留为 `baseKjCode`，不能静默唯一命中。 |
| KJ-编码（标准编码） | V1 主要标准 KJ，通常包含厚度、PA/BA、A/B、AT/MT 等完整规格信息。 | 优先作为 `standardKjCode`。 |
| 旧 KJ.NO | 孚盟 / 历史旧码。 | 保留为 `oldKjNo` / `fumacrmCode`，用于历史 CRM 或旧业务引用。 |
| 鼎捷编码（不带水箱盖） | ERP / 生产 / 下单编码之一。 | 保留为 `dingjieCodeWithoutCap`。 |
| 鼎捷编码（带水箱盖） | ERP / 生产 / 下单编码之一。 | 保留为 `dingjieCodeWithCap`。 |

### 匹配规则

1. 用户输入完整 KJ-编码，优先做标准 KJ 精确匹配。
2. 用户输入旧 KJ.NO，可按孚盟旧码匹配，但需要映射到标准 KJ。
3. 用户只输入基础编码，例如 `KJ-10001`，可能返回多候选。
4. 基础编码多候选时，系统不能静默自动选择第一行。
5. 鼎捷编码可以作为 ERP 编码保留和后续生产 / 下单使用，但 V1 不应只靠鼎捷编码静默报价。
6. 多编码冲突时，必须提示人工确认。

未来草稿候选中应保留字段：

- `standardKjCode`
- `baseKjCode`
- `oldKjNo`
- `fumacrmCode`
- `dingjieCodeWithoutCap`
- `dingjieCodeWithCap`

本轮不实现输出，只写规则。

## OEM / OE 规则

V1 不做 OEM 自动匹配。

原因：

1. 水箱 / 中冷器里的 OEM / OE 多在辅助 sheet 中。
2. 不一定和主成本行一一对应。
3. 旧 OEM 数据可能没有持续维护。
4. OEM / OE 未来应作为候选匹配 + 人工确认。

V1 规则：

1. 只支持 KJ 输入。
2. OEM / OE 输入标记为 `oem_not_supported_yet` 或 `requires_technical_review`。
3. OEM / OE 字段要预留。
4. 不使用水箱 / 中冷器 Excel 辅助 sheet 自动报价。
5. 未来 V2 再做 OEM → KJ 候选匹配。

## 规格 / 包装字段分类

### 产品规格字段

- 车系
- PA/BA
- 年份
- 书目
- 厚度 PA/BA
- 波距
- A/B
- AT/MT
- 车型
- 型号
- 规格
- 芯高
- 芯宽
- 芯厚
- 主板上
- 主板下
- 主板长
- 水室
- 主板
- 芯体

### 包装 / 体积字段

- 纸箱（内尺寸）长
- 纸箱（内尺寸）宽
- 纸箱（内尺寸）高

用途：

1. 包装体积计算。
2. 后续判断是否够柜。
3. 不等于产品主规格。

### 价格影响字段

- 特殊包装
- 客户品牌包装
- 中性包装
- 特殊标签
- 特殊唛头
- 其他包装要求

规则：

- 包装会影响价格。
- V1 可以先显示 warning，不自动计算特殊包装成本。
- 特殊包装及其他表不进入 V1 产品标准报价行，只作为包装 / 附加项候选。

## 风险字段规则

### 保留 warning，但 V1 不自动阻断

- 是否允限销
- 是否允限销备注

规则：

- 如果存在限销信息，V1 可生成 warning：`存在限销信息，请下单前确认。`

### V1 暂不作为自动阻断字段

- 能否报价
- 不能生产
- 不保质
- 漏水说明
- 风险说明
- 只做报价不公布

规则：

1. 如果这些字段来自主 sheet，可作为未来 warning 规则来源。
2. 如果数据来自特殊 sheet，例如 `不能生产` 或 `不保质漏水`，不进入 V1 主草稿。
3. V1 不静默自动报价。

## 价格字段规则

### 外销成本候选

字段：

- `2026.5.11出口成本报价`

含义：

- 外销 / 境外收美金 / 有退税场景的成本候选。

### 内销成本候选

字段：

- `2026.5.11出口部内销成本报价`

含义：

- 内销 / 收人民币场景的成本候选。

### 价格边界

1. 两列都只是成本候选 / `priceCandidate`。
2. 不是 `FinanceApprovedPrice`。
3. 不是正式报价。
4. 不能直接发客户。
5. 出口部不能根据这些字段自行决定底价、毛利或特殊折扣。

### 历史日期价格列

例如：

- `2026.4.10出口成本报价`
- `2026.4.10出口部内销成本报价`

规则：

1. V1 默认只取财务确认的最新有效日期列。
2. 历史日期列不作为 V1 默认报价候选。
3. 历史日期列未来可进入价格版本记录。
4. 不要一次性把所有历史列给出口部选择。

## 图片规则

Excel 嵌入图片不是稳定产品主图来源。

规则：

1. V1 不导入 Excel 嵌入图片。
2. V1 不把 Excel 图片上传 OSS。
3. V1 不把 Excel 图片当正式产品主图。
4. V1 可以显示 `imageStatus = embedded_only` 或 `missing`。
5. 未来需要建立 KJ → 默认主图 的统一产品图片库。
6. 图片主数据应由技术 / 产品资料侧维护。

## 数量字段规则

数量字段必须保留。

规则：

1. 业务员输入 KJ 时可以同时输入数量。
2. 数量可为空，但要 warning：`缺少数量`。
3. 数量存在时保存为 `quantity`。
4. 未来可结合纸箱尺寸计算体积 / 是否够柜。
5. V1 不一定实现体积计算，但 DTO 要保留 `quantity`。

## Dry-run warning 固化

水箱 / 中冷器 dry-run summary 应保留或输出以下 warning：

- 成本价不是财务批准价格。
- 报价表由财务提交和维护，出口部不能上传或维护报价表。
- 水箱 / 中冷器存在多编码、多规格、多包装字段，V1 需要人工确认。
- OEM / OE 自动匹配暂未开放。
- Excel 嵌入图片不是稳定主图来源。
- 特殊包装及其他只能作为包装 / 附加项候选，不能作为产品标准报价。

## 004B-R 行级人工确认口径

Quote Task 004B-R 修正口径：水箱 / 中冷器的 readiness 是 `v1_eligible_with_conditions`，不是整类 `v1_manual_confirmation_required`。

可直接进入 V1 草稿的情况：

- 输入完整 `KJ-编码（标准编码）`。
- 主成本表唯一匹配。
- 没有特殊 sheet、限销 / 风险说明、包装规格不明确、Excel 嵌入图等行级风险。

必须行级人工确认的情况：

- 输入基础 KJ，匹配多个候选。
- 输入旧 KJ.NO / 孚盟编码。
- 输入鼎捷编码。
- 输入 OEM / OE。
- 命中特殊 sheet，例如 `不能生产`、`只做报价，不公布的报价表`、不保质漏水。
- 命中限销 / 风险说明。
- 包装或特殊规格影响价格但信息不足。
- 图片只有 Excel 嵌入图。
- 找到多个候选行。

Go：

- 水箱 / 中冷器主成本表进入 V1 KJ 批量报价草稿方向。
- 完整标准 KJ 唯一命中时可进入 V1 草稿。
- 行级风险触发时输出人工确认 warning。

No-Go：

- 静默自动匹配基础 KJ。
- OEM / OE 自动匹配。
- 使用辅助 sheet 自动生成报价草稿。
- 把成本候选当作正式报价。
- 把 Excel 嵌入图当作产品主图。
- 让出口部上传或维护报价表。
