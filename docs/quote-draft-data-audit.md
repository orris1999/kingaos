# Quote Task 000｜出口部成本报价表结构盘点与数据字典

日期：2026-05-12

本文件只记录 2026 年 5 月 11 日出口部成本报价表的结构摘要、字段映射和后续报价草稿生成建议。报价表原始文件未复制进仓库，未导入数据库，未写入 seed，也未在本文档中展开成本价或报价明细。

## 结论摘要

- 8 个报价表均已完成只读结构盘点。
- 主生产类报价表普遍以 `KJ` 编码为核心，可支撑第一阶段 `KJ` 批量报价草稿。
- `OEM / OE` 字段只在部分 `不能生产`、`只做报价，不公布的报价表` 等 sheet 中出现，不适合第一阶段直接作为自动匹配主键。
- 多数表的价格字段命名为 `成本`、`成本报价`、`出口成本价`、`出口部内销成本价`，更像成本候选数据，不是财务批准价格。
- 图片信息在部分主表中以嵌入图片形式存在，但没有稳定的图片字段或图片 URL，暂不建议 V1 带图自动报价。
- 不同品类表结构不一致，尤其水箱 / 中冷器存在多个辅助 sheet、公式列和大量字段；需要先做标准化导入适配层。

## 字段映射建议

| 原始列名或等价列 | 建议字段 | 说明 |
|---|---|---|
| `KJ编码`、`KJ-编码`、`KJ.NO`、`KJ号`、`产品KJ编码`、`主件品名（KJ编码）` | `kjCode` | V1 主匹配字段；需设定优先级和规范化规则。 |
| `旧 KJ.NO`、`原KJ.NO`、`（孚盟） KJ-编码`、`（ERP） KJ-编码` | `alternateKjCode` | 作为 KJ 别名，不应直接覆盖主 KJ。 |
| `OEM`、`OE`、`原厂号` | `oemCode` | V2 再纳入；当前覆盖不均且重复风险高。 |
| `车型`、`车型车系`、`适用车型`、`英文车型`、`车系` | `vehicleModel` | 可用于报价草稿展示和人工核对。 |
| `产品`、`品名`、`名称`、`机冷品名`、`水室品名` | `productName` | 多数表没有标准产品名列，部分字段是部件名称。 |
| `车型类别`、`自产/外购`、`PA/BA`、`A/B`、`AT/MT` | `category` / `productAttribute` | 建议作为属性字段，不宜混入产品名称。 |
| `规格`、`芯体`、`芯体总成尺寸`、`波距`、`厚度`、`纸箱尺寸` | `spec` / `dimensions` | 结构差异较大，可先作为展示字段。 |
| `出口成本`、`出口成本价`、`出口成本报价` | `costPriceExport` | 成本候选，不是财务批准价。 |
| `出口部内销成本`、`出口部内销成本价`、`出口部内销成本报价` | `costPriceDomesticForExportDept` | 成本候选，不是正式报价。 |
| `报价`、`单价`、`新报价业务` | `quotedPriceCandidate` | 需人工确认含义，有些列名虽含报价但实为成本报价。 |
| `纸箱尺寸`、`包装`、`两台一包`、`吸塑` | `packaging` | 可用于报价草稿备注或包装方案。 |
| `能否报价`、`不能生产`、`是否允限销` | `quoteStatus` | 用于拦截或提醒，不应直接生成正式报价。 |
| `备注`、`是否允限销备注`、风险说明 sheet | `internalNote` | 内部备注，不应出现在对外报价。 |
| 嵌入图片 | `imageAssetCandidate` | 只检测到工作簿内图片，不是稳定 URL；V1 不建议自动带图。 |

## 关键字段覆盖判断

| 字段 | 识别情况 | 结论 |
|---|---|---|
| KJ 编号 | 大多数主表都有，水室唯一性最好；冷凝器、蒸发器、暖风、水箱、中冷器存在多个 KJ 变体列 | 可支撑 V1 KJ 批量报价草稿，但必须做 KJ 规范化和主列优先级 |
| OEM / OE 编号 | 主要出现在水箱 / 中冷器辅助 sheet，主成本 sheet 覆盖不足 | 暂缓自动匹配，先用于人工核对或 V2 |
| 产品名称 | 水室、机冷、特殊包装较明确；水箱 / 中冷器 / 冷凝器更多是车型 / 规格组合 | V1 可带展示名称，但需要生成规则 |
| 图片 | 冷凝器、水箱、中冷器检测到嵌入图片 | 不建议 V1 自动使用；需要独立资产抽取方案 |
| 价格 / 成本 | 多数是成本价、成本报价、出口部内销成本 | 只能作为草稿成本候选，不能作为财务批准价格 |
| 包装 | 多数主表包含纸箱尺寸，特殊包装表包含包装附加项 | 可带包装备注或附加项候选 |
| MOQ / 有效期 / 币种 | 未形成统一字段 | V1 不应自动承诺 |
| 状态 | 水箱 / 中冷器有能否报价、不能生产、限销相关字段 | 可作为草稿风险提醒 |

## 数据质量和自动化风险

1. **KJ 多列并存**：同一表内存在标准编码、旧编码、ERP 编码、孚盟编码、总编码等多个候选字段，必须先定义主键优先级。
2. **KJ 重复信号**：多表存在重复 KJ 或同一 KJ 多变体，不能简单按 KJ 直接取第一条。
3. **OEM 覆盖不均**：OEM 多在辅助 sheet，且存在一个 OEM 对多个 KJ 的风险。
4. **图片不是稳定字段**：检测到嵌入图片，但没有可导入的图片路径或图片 ID。
5. **成本和报价命名混用**：`成本报价` 容易被误解为对外报价，系统必须标记为成本候选。
6. **辅助 sheet 有敏感/不公开信息**：`只做报价，不公布的报价表`、不保质说明等不能直接给销售端公开。
7. **公式列存在**：水箱 / 中冷器辅助 sheet 有大量公式，直接读取值前要明确使用公式结果还是公式结构。
8. **品类结构不一致**：水室、特殊包装、机冷不是同一产品维度，不能强行用一个通用表结构吞掉全部差异。

## 能支持的自动化程度

| 能力 | 当前判断 |
|---|---|
| KJ 精确匹配 | 可以先做 V1，但要限定主表、主 KJ 字段和冲突处理 |
| OEM / OE 匹配 | 暂缓自动化，适合 V2 做候选匹配 + 人工确认 |
| 带出产品名称 | 可部分支持，很多表需要用车型 + 规格 + 品类生成展示名 |
| 带出图片 | 暂缓，当前图片不是稳定数据资产 |
| 带出价格 | 可带出成本候选，但不能作为正式报价 |
| 带出包装信息 | 可带出纸箱尺寸 / 特殊包装候选 |
| 状态拦截 | 可把不能生产、限销、风险备注作为草稿风险提醒 |

## 价格边界和禁止事项

这些报价表当前只能作为 `KJ / OEM 批量报价草稿生成器` 的数据来源候选。

禁止：

- 直接视为财务批准价格。
- 直接生成正式报价。
- 绕过 `FinancePricing`。
- 让销售直接获得底价 / 毛利决策权。
- 将 `只做报价，不公布的报价表` 直接开放给销售或客户。
- 将成本价、毛利、财务核价逻辑写入出口部客户档案。

下一阶段系统名称建议：`KJ / OEM 批量报价草稿生成器`。

不建议命名为：`自动正式报价系统`。
